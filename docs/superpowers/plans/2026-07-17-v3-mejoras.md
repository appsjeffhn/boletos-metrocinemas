### Task C3: Taquilla offline / tolerancia a mala señal (PWA)
> **IMPORTANTE (garantía anti-duplicidad):** el canje offline NO puede garantizar unicidad en el momento (dos dispositivos sin red podrían "canjear" el mismo boleto). Diseño: **validación cacheada + cola de canjes con reconciliación en el servidor** (el servidor es la fuente de verdad). Al reconectar, se envían los canjes encolados; si el servidor detecta que ya estaba canjeado, ese canje offline se marca **fallido/duplicado** y se alerta al operador. Confirmar este comportamiento con el usuario (Decisión #3).

**Files:** `public/sw.js` (ampliar), `src/lib/offlineQueue.ts` (IndexedDB), `src/components/Scanner.tsx`/`MultiScanner.tsx`, `src/app/canje/[token]/actions.ts`, nuevo endpoint `src/app/api/canje/sync/route.ts`, indicador de conexión en la UI de taquilla.
- [ ] Service worker: cachear el "app shell" de taquilla (rutas + assets) para que cargue sin red (cache-first para estáticos; nunca cachear respuestas de canje/auth).
- [ ] `offlineQueue.ts`: cola en **IndexedDB** de canjes pendientes `{ token, portadorNombre, portadorDni, sedeId, ts }`. Si el POST de canje falla por red, encolar y mostrar "Guardado sin conexión, se sincronizará".
- [ ] Sincronización: al reconectar (evento `online` y/o Background Sync), enviar la cola a `/api/canje/sync` que ejecuta `canjearBoleto` por cada uno (misma guardia atómica) y devuelve resultado por token; actualizar UI y **alertar los que quedaron duplicados/fallidos**.
- [ ] Indicador de estado (En línea / Sin conexión + N pendientes) en el header de taquilla.
- [ ] Tests del endpoint sync (reusa canjearBoleto) + de la lógica de cola (unit, mock IndexedDB o extraer lógica pura). `tsc`+`build`+`test`. Commit. Verificación manual con red desconectada.
